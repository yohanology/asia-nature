## Site Context

- **Type:** French-first bilingual marketing site (B2B sourcing agent, Shenzhen)
- **Primary goal:** organic visibility for French + English sourcing-agent intent
- **Likely priority keywords:** `agent sourcing Chine / Shenzhen`, `sourcing agent Shenzhen`, `sourcing industriel Asie`
- **Architecture:** Vite + React SPA, 2 routes (`/fr`, `/en`), single `Home` page
- **Deploy:** Vercel; primary domain `asia-nature.com` (pending DNS)

---

## Executive Summary

The on-page foundation is **strong** after today's fixes тАФ canonical, OG, hreflang, JSON-LD, and per-route meta are all in place and aimed at `asia-nature.com`. The DNS switch will not break indexation.

**Top priorities:**

1. ЁЯЯе **Image weight** тАФ 400тАУ600 KB JPGs ├Ч 10+. LCP + mobile will suffer. Convert to WebP with responsive `srcset`.
2. ЁЯЯз **SPA meta delivery** тАФ crawlers that don't execute JS see `index.html` fallback meta, not per-language meta. Affects `/fr` vs `/en` differentiation in Google's first-pass crawl.
3. ЁЯЯз **Thin information architecture** тАФ single page for all keywords. You're forfeiting long-tail ranking (services, FAQ, case studies).
4. ЁЯЯи **Font loading** тАФ `@import` in CSS blocks render. Move to preconnect + `<link>` in head.
5. ЁЯЯи **404 behavior** тАФ `vercel.json` rewrites everything to `/`, so unknown paths return 200 + content. Google treats as soft 404.

## Technical SEO

| Issue | Impact | Evidence | Fix |
|---|---|---|---|
| Large images, no WebP, no srcset | ЁЯЯе High | `public/asia_nature_bureau_shenzhen_1.jpg` = 593 KB; 6 JPGs > 400 KB; hero is `images.unsplash.com` at 2340w with no size hints | Convert to WebP (q80), generate 480/960/1920 widths, use `<picture>` or `srcset`+`sizes` |
| SPA soft-404 on unknown routes | ЁЯЯз Med | `vercel.json`: `{ "source": "/(.*)", "destination": "/" }` returns 200 for `/anything` | Add a real `/404` route returning 404 status, or use Vercel's `notFound` handling |
| No preconnect to external hosts | ЁЯЯз Med | Hero from Unsplash, fonts from Google, video from Vimeo тАФ no `rel="preconnect"` in head | Add `<link rel="preconnect">` for `fonts.googleapis.com`, `fonts.gstatic.com` (crossorigin), `images.unsplash.com`, `player.vimeo.com` |
| Render-blocking font `@import` | ЁЯЯи Low-Med | `src/index.css:2` uses `@import url('https://fonts.googleapis.com/...')` | Move fonts to `<link rel="stylesheet">` in `index.html` head with preconnect; `&display=swap` already present |
| www тЖФ apex strategy undefined | ЁЯЯз Med | Vercel shows both `asia-nature.com` and `www.asia-nature.com` as separate domains | In Vercel, set `asia-nature.com` as primary and 301 `www` тЖТ apex (matches canonical + sitemap) |
| HTTPS + HSTS | тЬЕ OK | `strict-transport-security: max-age=63072000; includeSubDomains; preload` тАФ excellent |
| `robots.txt` + `sitemap.xml` | тЬЕ OK | Both serve 200; sitemap references `asia-nature.com` |

## Prioritized Action Plan

### Quick wins (next 24h)

1. **Set primary domain in Vercel** тАФ `asia-nature.com` apex; `www` тЖТ 301 redirect.
2. **Compress images** тАФ e.g. `for f in public/*.jpg; do cwebp -q 80 "$f" -o "${f%.jpg}.webp"; done`, then swap `<img>` sources to `.webp`. Expect 50тАУ70% weight reduction.
3. **Add preconnect hints** to `index.html`:
   ```html
   <link rel="preconnect" href="https://fonts.googleapis.com" />
   <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
   <link rel="preconnect" href="https://images.unsplash.com" />
   ```
4. **Move Google Fonts from `@import` to `<link>` in head** with `display=swap`.
5. **Submit `sitemap.xml`** in Google Search Console after DNS flip. Add both `asia-nature.com` and `www.asia-nature.com` properties; set apex as primary; request validation.

### Medium-term (2тАУ4 weeks)

7. Build 2тАУ3 service subpages per language.
